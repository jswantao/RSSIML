## E1: CSI 单次认证实验

**科学问题**：基于 CSI 的单次身份认证能达到多高的精度？

### 参数设定

| 参数         | 值                         |
| ------------ | -------------------------- |
| 数据源       | CSI (NPY)，19 用户 (1-19)  |
| 模型         | SVM (在线 linear 核)       |
| 动作筛选     | 10 个选定动作              |
| 窗口大小     | 200，步长 100              |
| 测试比例     | 20%                        |
| 每用户文件数 | 100 (10 动作 × 10 trials)  |
| 降噪         | Butterworth 低通 (20Hz)    |
| 评估指标     | HTER, FAR, FRR, Accuracy   |

### 实验变体

仅文件级随机划分 (`cross_activity=False`)，训练与测试使用相同动作集。

### 实现

```python
# experiments/e1_csi.py — run_e1()
r = self._run_svm(
    cross_activity=False, seed=42, test_size=0.2,
    window_size=200, step_size=100, max_files_per_subject=100,
    threshold_method="youden", cv_folds=5,
    use_online_svm=True, online_kernel="linear",
    csi_denoise="butterworth",
)
```

### 输出

- 每用户 FRR vs FAR 柱状图 + 散点图
- 冒名攻击防御评级 (优秀/良好/一般/较差)
- 系统指标卡片 (HTER/FAR/FRR/准确率/F1)

## E2: RSSI 持续认证实验

**科学问题**：使用 RSSI 进行持续认证时，基础训练→推理阶段的全流程复现效果如何？

### 参数设定

| 参数     | 值                               |
| -------- | -------------------------------- |
| 数据源   | RSSI (MAT)，5 用户 (1-5)         |
| 模型     | SVM (批处理 RBF, class_weight=balanced) |
| 训练配置 | seed=42, test_size=0.2, ws=200, ss=100, cv_folds=5 |
| 认证方案 | 与推理阶段 `_render_continuous_auth` 完全一致 |
| 评估指标 | 接受率, 最长连续接受, 状态切换, 最终决策 |

### 实现

```python
# experiments/e2_rssi.py — run_e2()
# 阶段1: 训练 (匹配基础训练标签页 RSSI SVM 默认配置)
r = self._run_svm(
    seed=42, test_size=0.2, window_size=200, step_size=100,
    threshold_method="youden", cv_folds=5,
    use_pca=False, use_online_svm=False,
    feature_groups=("spectral", "statistical", "temporal"),
    cross_activity=False, clean_intermediate=True,
)

# 阶段2: 逐用户持续认证 (与推理阶段完全一致)
for subj in verifiers:
    raw = load_subject_raw(subj)           # 加载原始 .mat
    windows = build_windows(raw)           # ws=200, ss=100
    feats = extract_features(windows)      # 相同特征提取
    scores = svm_scores(verifier, feats)   # 逐窗口评分
    smoothed = moving_average(scores, 10)  # 前向滑动平均
    decisions = smoothed >= threshold      # 逐窗口判定
```

### 输出

- 每用户汇总表 (接受率、最长连续接受、状态切换、最终决策、训练 HTER)
- 代表用户 (最优/最差) 的推理风格分数曲线 + 决策条图
- 系统级指标卡片 (HTER/FAR/FRR/准确率)

| 窗口大小     | 200，步长 100 | 200，步长 100 |
| 测试比例     | 20% | 20% |
| 每用户文件数 | 100 | 100 |
| Epochs       | — | 10 |
| Batch size   | — | 96 |
| 架构         | — | conv=(32,64,128,192), hidden=256 |
| 梯度检查点   | — | 关闭 (~40% 提速) |
| 梯度累积     | — | 1 (直接更新) |
| 评估指标     | HTER, FAR, FRR, Accuracy + 训练耗时 |

## E5: RSSI 切片式单次认证实验

**科学问题**：切片时长如何影响单次认证判定的可靠性？

### 参数设定

| 参数     | 值                               |
| -------- | -------------------------------- |
| 数据源   | RSSI (MAT)，5 用户 (1-5)         |
| 模型     | SVM (批处理 RBF)                 |
| 切片时长 | [3, 5, 10, 20] 秒               |
| 训练配置 | seed=42, test_size=0.2, ws=200, ss=100, cv_folds=5 |
| 判定逻辑 | 与推理阶段切片模式完全一致          |
| 评估指标 | 切片通过率, HTER, FAR, FRR, 每用户分布 |

### 实现

```python
# experiments/e5_rssi_slice.py — run_e5()
# 阶段1: 训练 SVM (匹配基础训练)
r = self._run_svm(seed=42, test_size=0.2, ...)

# 阶段2: 多组切片时长对比 (匹配推理切片模式)
for dur in [3, 5, 10, 20]:
    for subj in verifiers:
        slices = slice_rssi(raw, slice_duration_s=dur)
        for seg in slices:
            windows = build_windows(seg)
            scores = svm_scores(verifier, feats)
            decision = (np.mean(scores) >= threshold)
```

### 输出

- 切片通过率 vs 切片时长折线图
- 切片粒度与样本量柱状图
- 每用户切片通过率分布
- 所有时长 × 用户的通过率热力图

## 数据流

```
PipelineConfig
  ├─ csi_selected_actions = (5,7,31,32,36,37,40,41,46,47)  # 更新后动作集
  ├─ SUBJECT_MAP_CSI  = {12→1, ..., 30→19}                  # 用户 ID 映射
  └─ SUBJECT_MAP_RSSI = {FXY→1, ..., ZX→5}

AuthPipeline(seed=42, test_size=0.2, window_size=200, ...)
  ├─ DatasetSplitter → 动作筛选 + 用户映射 + 文件级划分
  ├─ DataCoordinator.build_windows_memmap → 滑动窗口 + 可选降噪
  ├─ SVMAuthenticationTrainer → 在线 SVM (SGD+class_weight) / 批处理 SVC (RBF)
  └─ CNNTrainer → 1D-CNN (AMP + BCE pos_weight + HTER 早停)
```

### 类别不平衡处理

| 模型 | 惩罚机制 | 采样策略 |
|------|---------|---------|
| SVM 在线 | `class_weight={0:w0, 1:w1}` 预计算 | 负样本下采样 max(5000, n_pos×5) |
| SVM 批处理 | `class_weight='balanced'` | 同上 |
| CNN | `BCEWithLogitsLoss(pos_weight=n_neg/n_pos)` | 同上 |
